# 蛋白质语言模型（PLM）序列理解 — 解读笔记

> 返回：[← 01_protein_biology.md](../01_protein_biology.md)

---

## 这一节在问什么？

**PLM 之于蛋白质，就像 BERT/GPT 之于文本——但它们真的"理解"蛋白质序列了吗？**

PLM（Protein Language Model）把氨基酸序列当成"文字"来训练，代表模型有 ESM2、ProtT5、Ankh 等。它们在数十亿条蛋白质序列上做自监督预训练（类似 BERT 的 Masked Language Modeling），学完之后可以给任意蛋白序列生成一个"表征向量"（embedding）。

---

## 为什么还需要专门评测它？

> **预训练得好 ≠ 下游任务都好用。**

GPT 的 perplexity 低，不代表它在法律文本分类、医学 NER 上都最强——同理，PLM 的预训练 loss 低，不代表它对蛋白质的每一种生物学属性都能理解。

这一节的 benchmark 专门测：**把 PLM 的 embedding 拿来做各种下游任务，它到底有多强？**

---

## 两个 Benchmark 具体测什么

### 多任务 PLM 评测（Nat. Commun. 2024）

把同一组 PLM（ESM2、ProtT5、Ankh 等）在 **6 类任务** 上横向对比：

| 任务 | 生物学意义 |
|------|-----------|
| 二级结构预测 | 每个氨基酸是 α-螺旋 / β-折叠 / 无规则？ |
| 内在无序区预测 | 哪些区段没有固定结构？ |
| 亚细胞定位分类 | 这个蛋白在细胞哪里工作？ |
| 热稳定性预测 | 加热到多少度才失活？（回归） |
| 溶解度预测 | 在大肠杆菌里会不会形成包涵体？ |
| 表位预测 | 哪些位点会被抗体识别？ |

**核心结论**：所有任务上"微调必须做"——零样本直接用 embedding 效果差，说明 PLM 的通用表征还不够强到开箱即用。

---

### DARKIN（2024）

专门测一个更难的问题：**"黑暗激酶"（dark kinases）**。

这类激酶在文献里极少被研究，几乎没有实验标注数据。测的是：PLM 在**零样本条件**下，能否仅凭序列相似性识别这些激酶的磷酸化位点？

**核心结论**：最好的 PLM 零样本 AUC ~0.75，监督微调可达 ~0.88，零样本迁移能力仍有约 10% 的明显差距。

---

## 和前几节（CASP / ProteinGym）的区别

```
CASP         → 测"结构预测"这一件事做得有多准
ProteinGym   → 测"突变适应性打分"这一件事做得有多准
PLM 多任务评测 → 测"模型的通用表征"能支撑多少种下游任务
```

**NLP 类比**：

| 蛋白质领域 | NLP 类比 |
|-----------|---------|
| CASP | SQuAD（专门测阅读理解） |
| ProteinGym | GLUE 某一项子任务 |
| PLM 多任务评测 | **GLUE / SuperGLUE**（测语言模型的通用能力上限） |

---

## 对本项目的启示

我们的数据集里有 `<protein>` 标记。这一节的结论直接表明：

> **现有 PLM 在零样本迁移上还不够强，而且没有统一的跨任务评测体系。**

这正是新 benchmark 可以填补的空白之一：**序列标记（`<mol>` / `<protein>` / `<rna>`）的联合理解能力评测**，目前无成熟竞品。
